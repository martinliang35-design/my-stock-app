本目录存放项目的需求说明与技术设计文档，便于后续维护与迭代。

| 文档 | 说明 | 适合读者 |
|------|------|----------|
| [需求说明 (requirements.md)](./requirements.md) | 产品目标、功能清单、使用场景、迭代规划 | 产品、测试、新成员了解业务 |

## 文档维护建议

- **需求变更**：新增/修改功能时，先更新 `requirements.md` 中对应模块与「迭代规划」。
- **技术变更**：改架构、接口、表结构时，同步更新 `technical.md`，并注明变更原因与日期。
- **版本记录**：在各自文档末尾或单独 CHANGELOG 中记录重要版本与变更摘要。
