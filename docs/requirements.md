# Stock-Cloud 需求说明文档

## 1. 项目概述

### 1.1 项目名称与定位

- **项目名称**：Stock-Cloud（股票持仓看板）
- **定位**：个人多市场股票持仓管理 Web 应用，数据云端存储，支持 A 股、港股、美股。

### 1.2 目标用户与场景

- **目标用户**：个人投资者，需要在一个页面查看并管理 A 股、港股、美股的持仓与汇总。
- **典型场景**：
  - 注册/登录后进入个人看板，仅查看和管理**本人**持仓。
  - 查看持仓总市值、总盈亏、盈亏比例（按人民币折算）。
  - 手动维护持仓列表（代码、名称、市场、数量、成本价、现价）。
  - 一键刷新所有持仓的当前价格。
  - 通过饼图/柱状图查看资产分布与 Top 持仓。
  - 查看单只股票的 K 线走势（A 股/港股）。
  - 导入/导出 CSV 或 JSON，便于备份或迁移。

### 1.3 核心价值

- 多市场统一展示，汇率折算为人民币。
- 数据存于 Supabase，多设备可访问，无需本地安装。
- 行情与汇率通过自有 API 代理，国内可直接使用（不依赖需翻墙的服务）。

## 2. 功能清单

### 2.1 账户与登录

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 用户注册 | 邮箱 + 密码注册，写入 Supabase Auth | P0 |
| 用户登录 | 邮箱 + 密码登录，会话持久化（如 localStorage） | P0 |
| 登出 | 退出登录，清除会话 | P0 |
| 多用户数据隔离 | 持仓表按 user_id 隔离，仅能查看/编辑本人数据（RLS） | P0 |

### 2.2 持仓管理

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 持仓列表展示 | 表格展示：代码、名称、市场、数量、成本价、现价、市值、盈亏、盈亏比例等 | P0 |
| 新增一行 | 在表格中新增一条空持仓记录，可编辑后保存 | P0 |
| 编辑持仓 | 支持修改代码、名称、市场、数量、成本价、现价；失焦或选择变更时落库 | P0 |
| 删除持仓 | 单行删除，确认后从数据库移除 | P0 |
| 拖拽排序 | 支持拖拽行调整顺序，顺序持久化到数据库 | P1 |

### 2.3 行情与汇率

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 刷新股价 | 按当前列表逐只请求行情接口，更新「现价」并写回数据库 | P0 |
| 汇率配置 | 美元/港币对人民币汇率可手动输入；可选从接口拉取并覆盖 | P0 |
| 汇率持久化 | 汇率存于浏览器 localStorage，刷新页面后保留 | P1 |
| 自动刷新 | 可选定时自动刷新股价（间隔可配置），偏好存 localStorage | P2 |

### 2.4 汇总与展示

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 汇总卡片 | 展示总市值、总成本、总盈亏、总盈亏比例（人民币口径） | P0 |
| 市值占比饼图 | 按市场（A/港/美）或按标的汇总市值占比 | P1 |
| Top 持仓柱状图 | 按市值取前 N 只的柱状图展示 | P1 |
| 明细列显隐 | 可选显示/隐藏部分明细列，便于小屏或精简视图 | P2 |

### 2.5 K 线

| 功能 | 描述 | 优先级 |
|------|------|--------|
| K 线弹窗 | 点击某行可打开弹窗，展示该标的 K 线图（日 K） | P1 |
| 支持市场 | 当前支持 A 股、港股；美股 K 线暂不支持 | P1 |

### 2.6 导入导出

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 导出 JSON | 导出当前持仓列表 + 汇率，便于备份或迁移 | P0 |
| 导出 CSV | 导出当前持仓为 CSV，便于在 Excel 中查看或编辑 | P0 |
| 导入 JSON | 选择此前导出的 JSON 文件，解析后**替换**当前持仓并写入数据库，可选写入汇率 | P0 |
| 导入 CSV | 选择 CSV 文件，解析后**替换**当前持仓并写入数据库；表头需与导出格式兼容 | P0 |

## 3. 非功能需求

| 类型 | 说明 |
|------|------|
| 兼容性 | 支持现代浏览器（Chrome、Edge、Safari、Firefox 等）。 |
| 数据与安全 | 持仓存 Supabase，按 user_id 行级隔离（RLS）；未登录仅可访问登录/注册页，登录后仅能读写本人持仓。 |
| 行情与汇率 | 行情、汇率均通过 Next.js 服务端代理请求第三方，避免前端跨域；国内不依赖需翻墙的接口。 |
| 性能 | 刷新股价为逐只请求，数量较多时耗时会增加，后续可考虑批量接口或限流提示。 |

## 4. 术语与约定

| 术语 | 说明 |
|------|------|
| 市场 | A = A 股，HK = 港股，US = 美股。 |
| 市值 | 现价 × 数量；港股/美股按配置汇率折算为人民币后参与汇总。 |
| 成本 | 成本价 × 数量；折算规则同上。 |
| 盈亏 | 市值 − 成本；盈亏比例 = 盈亏 / 成本 × 100%。 |
